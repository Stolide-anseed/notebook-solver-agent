# notebook-solver-agent

## Описание проекта

`notebook-solver-agent` — это AI-система для автоматизации работы с университетскими тетрадками и учебными заданиями.
Проект предназначен для того, чтобы избавить пользователя от монотонной ручной работы: чтения PDF-тетрадок, поиска нужной теории, разбора заданий и переноса решений в аккуратный формат.

Система должна принимать тетрадку в формате PDF, извлекать из неё задания и теоретический материал, анализировать содержание, последовательно решать задачи и формировать итоговый результат в формате Jupyter Notebook (`.ipynb`).

## Цель проекта

Главная цель проекта — создать agent-based систему, которая сможет:

* читать учебные тетрадки и задания из PDF,
* выделять теорию и отдельные задачи,
* использовать как материал из самой тетрадки, так и внешние источники,
* решать задания по порядку,
* автоматически оформлять ответы в Jupyter Notebook.

Идея проекта состоит в том, чтобы убрать рутинную часть учебного процесса и автоматизировать повторяющиеся действия, оставив человеку контроль, проверку и доработку результата.

## MVP

Первая рабочая версия системы должна уметь:

* загружать PDF-документ,
* извлекать из него текст,
* находить задания и связанный с ними теоретический контекст,
* последовательно обрабатывать задачи,
* генерировать решения и пояснения,
* сохранять результат в формате `.ipynb`.

## Полное видение

В полной версии проект должен представлять собой end-to-end систему, которая работает по следующему сценарию:

1. Пользователь загружает тетрадку в PDF.
2. Система извлекает и структурирует содержимое документа.
3. Агент определяет, где находится теория, а где — задания.
4. Для решения используются:

   * данные из самой тетрадки,
   * локальная база знаний,
   * внешние источники и поиск в интернете при необходимости.
5. Задания решаются последовательно.
6. Итог оформляется в Jupyter Notebook с текстом, формулами, пояснениями и кодом, если это требуется типом задачи.

## Практическая ценность

Проект решает понятную прикладную задачу: значительная часть учебной работы состоит из однотипных и рутинных действий, которые занимают много времени, но не всегда дают реальную пользу для понимания материала.
Такую работу можно частично передать AI-системе, чтобы сосредоточиться на более важной части — проверке, осмыслении и доработке решений.

## Техническая ценность проекта

Этот проект также важен как инженерная и исследовательская задача.
В процессе его разработки можно глубоко изучить и применить на практике:

* Transformers,
* Retrieval-Augmented Generation (RAG),
* обработку и парсинг документов,
* agent workflows,
* LangGraph,
* генерацию и структурирование notebook-файлов.

## Ограничения первой версии

На старте проект не должен пытаться решать всё сразу.
Первая версия может быть ограничена следующими рамками:

* работа только с печатными или хорошо читаемыми PDF,
* базовое извлечение текста,
* простой поиск релевантного контекста,
* генерация чернового решения,
* экспорт результата в `.ipynb`.

Более сложные возможности можно добавить позже:

* OCR для рукописных тетрадок,
* multi-agent архитектуру,
* модуль проверки решений,
* использование внешних инструментов для вычислений,
* продвинутый интернет-поиск,
* адаптацию под разные предметные области.

## Ожидаемый результат

Результатом работы системы должен быть Jupyter Notebook (`.ipynb`), который содержит:

* условия задач,
* нужную теорию или краткие пояснения,
* ход решения,
* формулы,
* кодовые ячейки при необходимости,
* финальные ответы в аккуратном и понятном виде.
